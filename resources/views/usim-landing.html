Server-Driven UI · Laravel · v0.7.0

Define tu UI
en PHP puro.
El resto es automático.

USIM es un framework Server-Driven UI para Laravel. Define pantallas, formularios, tablas y modals en PHP — el framework se encarga de renderizar, differenciar y actualizar todo en el cliente automáticamente.

v0.7.0 Versión actual
MIT Licencia libre
PHP 8.2+ Requerido
HelloScreen.php 
class HelloScreen extends AbstractUIService
{
  protected function buildBaseUI(
    UIContainer $container
  ): void {
    $container->layout(LayoutType::VERTICAL)
             ->padding(20);

    $container->add(
      UIBuilder::label('title')
        ->text('Welcome to USIM!')
        ->style('h1')
    );

    $container->add(
      UIBuilder::button('hello_btn')
        ->label('Say Hello')
        ->primary()
        ->action('greet')
    );
  }

  public function onGreet(array $params): void
  {
    $this->toast('Hello, USIM!', 'success');
  }
}
Instalar via Composer:
composer require idei/usim
php artisan usim:install

Características

¿Por qué USIM?

Un nuevo paradigma donde el servidor controla la UI y el cliente sólo renderiza el resultado.

🖥️

Server-Driven UI

Define toda tu interfaz en PHP. Pantallas, menus, formularios, tablas y modals sin escribir una sola línea de JavaScript.

⚡

Actualizaciones Delta

El framework calcula la diferencia entre el estado anterior y el nuevo, enviando al cliente sólo los cambios necesarios.

🎯

Sistema de Eventos

Manejo de eventos con convención de nombres automática. Botones, formularios y acciones se conectan solos al handler correspondiente.

🔐

Auth Completo

Login, registro, recuperación de contraseña, perfiles y gestión de roles con Sanctum y Spatie Permission incluidos.

🧪

Testing Integrado

Guías y helpers para testear screens con patrones claros, assertions por componente y soporte para agentes de IA generadora de tests.

🚀

Octane / RoadRunner

Compatible con procesos de larga duración. Reset automático del generador de IDs por request, sin configuración extra.

Arquitectura

Client-Agnostic por diseño

El servidor genera un contrato JSON agnóstico. Cualquier cliente — web, React, Android — puede implementar su propio render.
